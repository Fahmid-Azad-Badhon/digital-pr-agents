# Pitch Angles

## Stage Contract
- Required Input: `02-insights.md` and `03-research.md`.
- Optional Input: user angle preference, geography, competitor coverage, seasonal relevance, local/county/state data.
- Output: 40 pitch angles (20 categories x 2 angles) with category, journalist beats, pitch angle, 250-300 word "Why This is Newsworthy" justification, score (1-10), and checkbox for selection.
- Handoff: send to GPT-5.5 for review/scoring, then to user for selection.
- Validation: every pitch angle must be unique, data-supported, easy to understand, newsworthy, beat-aligned, and ready for outreach scoring.
- Selected Angle Status: pending until user selects via checkbox.
- Source Rule: use SERP support where needed; do not invent statistics, studies, or media context.

## Newsworthiness Scorecard
Score every angle from `1-10` before it can move to Stage 05.

| Angle / Pitch Angle | Timeliness | Novelty | Data Strength | Human Impact | Local Relevance | Conflict / Tension | Public Interest | Journalist Beat Fit | Source Credibility | Emotional Resonance | Clear Editorial Reason | Headline Support | Average Score | Decision |
|---------------------|------------|---------|---------------|--------------|-----------------|--------------------|-----------------|---------------------|--------------------|---------------------|------------------------|------------------|---------------|----------|
|  |  |  |  |  |  |  |  |  |  |  |  |  |  |  |

Decision rule:
- `8.0-10`: may move to Stage 05.
- `6.5-7.9`: revise, strengthen, or move to secondary backlog.
- `1.0-6.4`: reject before journalist targeting.
- No angle may move to final email drafting unless it scores `8/10` or higher and has source-backed evidence.

## Anti-Hallucination And Assumption Control
Never invent journalist names, journalist emails, article titles, publication history, source names, statistics, rankings, survey results, methodology details, quotes, SERP findings, or Muck Rack results.

If required information is missing, unverifiable, blocked by access, or not present in the validated inputs, write exactly:

`Information unavailable. Verification required before use.`

Clearly label beat-level assumptions as assumptions. Do not present SERP opportunity, likely search demand, or possible Muck Rack discovery as verified results unless captured evidence exists in the job folder.
